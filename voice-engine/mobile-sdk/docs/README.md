# ai-engine 文档目录

本目录只放项目说明、产品设计、架构设计和外部资料。工程入口、依赖、构建产物和临时文件不放在这里。

## 阅读顺序

1. [实时语音与豆包语音合作设计](architecture/realtime-voice-doubao-cooperation-design.md)
   - 用于理解当前实时语音主链路、豆包语音能力拆分、推荐架构、长期记忆设计和阶段路线。
   - 适合研发、方案设计和对外合作前阅读。

2. [豆包语音陪伴类产品调研与落地设计](product/voice-companion-product-research.md)
   - 用于理解陪伴式语音 Agent 的产品方向、MVP 范围、场景模板、合规边界和商业化路径。
   - 适合产品规划、场景评审和试点方案讨论前阅读。

3. [AI 播客化与文档播报玩法调研](product/ai-podcast-product-research.md)
   - 用于理解“题目 / 文档 / 会议转写 -> 播客 / 播报”的产品玩法、MVP 边界、质量要求和落地顺序。
   - 适合规划长音频生成、知识库音频化、会后复盘音频和可追问播客前阅读。

4. [音色场景与设置设计](product/voice-scene-settings-design.md)
   - 用于理解当前代码中 O2.0 / SC2.0 音色、提示词、场景和高级参数的配置边界。
   - 适合新增场景、调整音色和设计运营配置前阅读。

5. [实时语音打断体验整理](product/realtime-interrupt-experience.md)
   - 用于理解当前通话中的按钮打断、语音打断、静音和结束通话边界。
   - 适合调整实时语音用户体验、补打断逻辑测试或改动前后端状态流前阅读。

6. [AI 与现实产品 / 硬件绑定深度调研报告](product/real-product-ai-binding-research.md)
   - 用于理解陪伴音响、品牌商品、IP 周边、成人亲密设备等现实产品如何绑定 AI 语音 Agent。
   - 适合评估硬件合作、商品扫码激活、设备控制权限、成人设备边界和产品绑定 MVP 前阅读。

7. [豆包原始资料](references/doubao/)
   - 存放从火山引擎/豆包语音获取的 PDF、音色表和接入资料。
   - 只作为依据材料归档，不在这里直接写项目方案。

## 目录约定

| 目录 | 内容 |
| --- | --- |
| `architecture/` | 架构设计、技术方案、接口边界、演进路线 |
| `product/` | 产品调研、PRD、场景设计、试点方案 |
| `references/` | 外部厂商文档、原始资料、调研依据 |

## 维护规则

- 新增 Markdown 文档时，先判断属于产品、架构还是参考资料，再放入对应目录。
- 外部 PDF、截图、导出资料放入 `references/<来源>/`，不要放在项目根目录。
- 文档内容尽量写清楚适用阶段和不做事项，避免把未来设想误当成当前范围。
